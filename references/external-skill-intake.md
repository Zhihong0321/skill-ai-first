# External Skill Intake

Use this reference when a stage would benefit from an outside `SKILL.md`, checklist, or workflow.

## Goal

Borrow useful patterns without importing outside priorities blindly.

## Intake Rules

1. identify the exact problem the external skill solves
2. extract heuristics, guardrails, or scripts that match this bundle's goals
3. rewrite those ideas into local references or local scripts
4. keep the single-task rule and staged workflow intact

## Do Not

- copy external skill bodies wholesale
- inherit conflicting priorities like speed over caution
- increase bundle complexity just because the outside skill is detailed

## Good Reuse

- file-splitting heuristics
- dead-code detection checklists
- rollout-safe cleanup patterns
- architecture review prompts

## Bad Reuse

- generic boilerplate that does not fit this repo
- multi-stage workflows that encourage rushing
- instructions that create more context noise than clarity
